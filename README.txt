AutoRia clone

Метою цього проєкту є створення бази данних для створення оголошень з продажу автомобілів та всіх відповідних ендпоінтів для роботи з нею.

Вимоги для запуску проєкту:
1. Версія Python 3.12. Якщо ви бажаєте її змінити, то Вам необхідно зробити відповідні зміни в файлах pyproject.toml та DockerFile.
2. Встановлений Docker та Poetry.
3. В якості Interpreter я використовую Poetry Interpreter, але за бажанням можна використовувати і Docker Interpreter.

*Всі команди розраховані на операційну систему Windows.*

Установка проєкту:
1. Клонувати проєкт з GitHub.
2. Налаштувати Interpreter.
3. Налаштувати Project Structure в налаштуваннях Вашого середовища. В якості Source Folder використовується папка backend. В якості Template Folders використовується папка backend/templates.
4. Запуск проєкту здійснюється через команду в терміналі: docker compose up --build.
5. Проєкт повністю не запуститься, поки не буде з'єднання з базою данних. Дані для з'єднання з базою данних знаходяться в файлі .env. В якості бази данних в проєкті використовується MySQL. Це можна змінити в backend/settings. Але відповідні зміни мають відбутись і в .env.
6. Виконати міграції для оновлення бази данних. Для цього декілька простих кроків:
    - доступитися до папки backend за допомогою команди в нове вікно терміналу: cd backend.
    - ввести наступні команди послідовно: python manage.py makemigrations та python manage.py migrate.
7. Ви запустили проєкт.

В якості підвищення безпеки замінити дані для під'єдння до бази данних.
В базі данних вже створений superuser. Email: superuser@gmail.com. Password: admin.
Ви можете видалити цього головного користувача з бази данних та створити нового за допомогою команди в термінал python manage.py createsuperuser. Передайте логін та пароль.

Коротка інформація про побудову проєкту:
    - Існує 3 головних apps(backend/apps): auth - логінація, активація користувача та оновлення паролю; create_car_ad - створення оголошення про продаж авто та всі взаємодії з ним; users - регістрація користувача та методи взаємодії з ним.
    - backend/configs - головна папка з налаштуваннями проєкту та всіх доданих бібліотек.
    - backend/core_app - спільна app з допоміжними методами, які можуть бути використані в усіх apps.
